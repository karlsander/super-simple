# Sample Library Notes

The app now exposes two drum source types:

- `electronic`: built-in synthesized voices for 808 / 909-style coverage
- `world`: one bundled sampled pack sourced from the local Splice library

The sampled pack is generated by:

- `Scripts/sync_splice_samples.py`

That script:

- copies the curated Splice one-shots into `SuperSimple/Resources/Samples/world`
- rewrites `SuperSimple/Resources/Samples/sample-library.json`
- rewrites `Docs/sample-library-manifest.json`

The manifest records, for every bundled sample:

- the bundled resource path
- the original relative path inside the local Splice library
- the filename credit
- optional mapping notes where a closest-match substitute was used

The count click is not part of the sampled pack. It always uses the built-in electronic click so the metronome sound stays constant across pack changes.

Current known compromises:

- `guache` is mapped to a shaker substitute because no true guache or cabasa was present
- `crossStick` is mapped to the closest available wood-click sample because no explicit cross-stick was present
